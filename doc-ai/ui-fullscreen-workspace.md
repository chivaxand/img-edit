## Core Concepts

Some advanced image processing features (such as interactive segmentation, 3D color cubes, normal/displacement map previews, and forensic split-panes) require substantial visual space. Standard small popups cannot comfortably accommodate these multi-canvas diagnostic workflows.

To address this, the `FullScreenWorkspace` overlay occupies the entire browser viewport (`100vw`, `100vh`) with a high `z-index`, sitting above the primary application editor interface. It provides a standardized double-pane workstation layout while remaining fully compatible with the custom programmatic controls in the core `UI` builder.

---

## Workspace Structure & CSS Layout

The workspace enforces a consistent CSS layout structure to preserve application theme cohesion:

- **Header Bar (`.fs-workspace-header`)**: Sits at the top of the viewport. Displays the tool title and primary action buttons: "Cancel" (discards changes) and "Apply" (commits the modification back to the active layer).
- **Control Sidebar (`.fs-workspace-sidebar`)**: A left-hand column dedicated to parameters. It handles standard form elements generated by `UI.createSliderRow()`, `UI.createSelectRow()`, and `UI.createCheckbox()`.
- **Content Viewport (`.fs-workspace-content`)**: A flexible, hardware-accelerated right-hand viewport container meant for custom canvas nodes, split-panels, or webgl comparison panels.

---

## State Retention & Lifecycle Patterns

When invoking a full-screen interactive workspace, developers must follow strict state management rules to prevent memory leaks and undo-stack corruption:

- **Clean Source Isolation**: Always read the original, clean pixel array from the active layer to feed the solver or algorithm. Do not let user drawing strokes/markup on the viewport canvases contaminate the source data.
- **Save State Prior to Apply**: When the user clicks "Apply", call `App.actions.saveState()` *first* so that the application history stack registers the correct "Before" state before the newly generated pixels are written into the target layer.
- **Escape Key Safety**: The workspace listens to `keydown` escape events globally. Always ensure that pressing Escape automatically triggers the same cleanup logic as clicking "Cancel".
- **DOM and Listener Clean-up**: On close, the overlay element must be removed from the document root, and all key/window listeners must be explicitly detached to release active memory.

## Implementation Template

Below is the standard programmatic recipe for implementing a full-screen interactive filter or custom tool:

```typescript
import { App } from '../app';
import { UI } from '../ui';

export const MyComplexWorkspaceFilter = {
    open() {
        const layer = App.utils.getActive();
        if (!layer) return alert("Select an active layer.");

        // 1. Initialize Workspace Layout
        const ws = new App.FullScreenWorkspace({
            title: "My Complex Workspace Title",
            onApply: () => {
                // Save history state first
                App.actions.saveState();
                
                // Write processed pixels back to the active layer
                const ctx = layer.canvas.getContext('2d')!;
                // ctx.putImageData(...)
                
                App.emit('layer:content'); // Refresh thumbnails
            },
            onCancel: () => {
                console.log("Workspace closed, no changes made.");
            }
        });

        // 2. Populate Sidebar Controls
        ws.sidebar.appendChild(UI.createNode('div', { className: 'fs-workspace-section-title' }, 'Parameters'));
        
        ws.sidebar.appendChild(UI.createSliderRow({
            label: 'Intensity', min: 1, max: 100, value: 50,
            onInput: (v) => {
                // Update local state and trigger preview
            }
        }));

        // 3. Populate Viewport Panels
        const panelGrid = UI.createNode('div', { style: 'display:flex; width:100%; height:100%; padding:15px; gap:15px;' });
        const liveCanvasObj = UI.createCanvas({ width: layer.canvas.width, height: layer.canvas.height });
        
        panelGrid.appendChild(liveCanvasObj.element);
        ws.content.appendChild(panelGrid);

        // 4. Mount / Show Workspace
        ws.show();
    }
};
```
